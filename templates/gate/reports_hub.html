What is Reports Hub?

Reports Hub is your control center for gate and attendance data. Everything here comes from other parts of the system:

  • Gate entries — who was granted or denied at the gate (see sidebar: Gate entries)
  • Denied entries — entries denied at the gate (see sidebar: Gate entries, filter by Denied)
  • Events — event-based scans and live counts (see sidebar: Events and Compare events)
  • Scanner devices — tablets/phones used at the gate; if they’re offline, new scans won’t show in Gate entries
  • Audit log — who did what in the system (same as sidebar: View logs)
{# ----- Report types: clear tabs with short descriptions ----- #}
Report types

Choose how you want to see data: pre-built summaries, custom date ranges, or event comparisons.

Generated reports On-demand report Compare events Audit log
  • Generated reports — Pre-made daily/weekly/monthly summaries (from Gate entries).
  • On-demand report — Build a custom report by date range and type (uses Gate entries, Events).
  • Compare events — Compare attendance or issues across two events (uses Events + Gate entries).
  • Audit log — Full history of system actions (same as View logs in the sidebar).

{# ----- Scheduled reports (simplified) ----- #}
Scheduled reports

These reports are created by running commands (e.g. via Task Scheduler or cron). They use the same Gate entries data you see in the menus above.

  • Daily — Entry summary, peak times, denied scans. Command: python manage.py generate_daily_report
  • Weekly — Week trends, top hours. Command: python manage.py generate_weekly_report
  • Monthly — Executive summary, scan success rate. Command: python manage.py generate_monthly_report
